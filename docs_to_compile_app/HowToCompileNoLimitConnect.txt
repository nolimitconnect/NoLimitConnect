
=== Windows Compile In Visual Studio ===

Compile In Visual Studio
In Visual Studio open nolimitapp/nolimitgui/build/VS2022/NoLimitConnectAll.sln

Depending on your preference do one of the following for Android Or Linux 

=== Compile in Qt Creator ===
Code Checkout
git clone https://gitlab.com/nolimitconnectapps/nolimitapp.git

In Qt Creator open nolimitapp/CMakeList.txt
Configure and Compile


=== Compile from command line in Linux ===

Export path to desired qt version
example for linux builds: 
export Qt6_DIR="~/Qt/6.6.2/gcc_64/lib/cmake/Qt6"
example for android aarch64 builds: 
export Qt6_DIR="~/Qt/6.6.2/android_arm64_v8a/lib/cmake/Qt6"
example for android arm7a builds: 
export Qt6_DIR="~/Qt/6.6.2/android_armv7/lib/cmake/Qt6"

Code Checkout
git clone https://gitlab.com/nolimitconnectapps/nolimitapp.git

cd nolimitapp/build
mkdir nlc_build
cd nlc_build
rm -rf *


If compiling for Linux
cmake -S ../.. -B . -DCMAKE_BUILD_TYPE=Release -DTARGET_OS_LINUX=true, -DQT_VERSION=6_6_2
or
cmake -S ../.. -B . -DCMAKE_BUILD_TYPE=Release -DTARGET_OS_LINUX=true, -DQT_VERSION=6_4_3

If cross compiling for Android in Linux
For 64bit arm cpu aarch64 (arm64-v8a)
cmake -S ../.. -B . -DCMAKE_BUILD_TYPE=Release -DTARGET_OS_ANDROID=true, -DQT_VERSION=6_6_2 -DARM64
or
cmake -S ../.. -B . -DCMAKE_BUILD_TYPE=Release -DTARGET_OS_ANDROID=true, -DQT_VERSION=6_4_3 -DARM64

For 32bit arm cpy arm7 (armeabi-v7a)
cmake -S ../.. -B . -DCMAKE_BUILD_TYPE=Release -DTARGET_OS_ANDROID=true, -DQT_VERSION=6_6_2 -DARM32
or
cmake -S ../.. -B . -DCMAKE_BUILD_TYPE=Release -DTARGET_OS_ANDROID=true, -DQT_VERSION=6_4_3 -DARM32


make -j$(nproc)

This may have been resolved. I have opened a bug QTBUG-119061

NDK KIT 23.1.7779620
SDK PLATFORM 33
ANDROID PLATFORM android-23
