


Depending on your preference do one of the following for Android Or Linux 

=== Compile in Qt Creator ===

Code Checkout
git clone https://gitlab.com/nolimitconnectapps/nolimitapp.git

In Qt Creator open nolimitapp/CMakeList.txt
Configure and Compile


=== Compile from command line in Linux ===

Code Checkout
git clone https://gitlab.com/nolimitconnectapps/nolimitapp.git

mkdir nolimitapp/build
cd nolimitapp/build
mkdir nlc_build
cd nlc_build
rm -rf *

To configure build for Linux:
cmake -S ../.. -B . -DCMAKE_BUILD_TYPE=Release -DTARGET_OS_LINUX=true, -DQT_VERSION=6_6_3


To configure build for 64bit arm cpu aarch64 (arm64-v8a):
cmake -S ../.. -B . -DCMAKE_BUILD_TYPE=Release -DTARGET_OS_ANDROID=true, -DQT_VERSION=6_6_3 -DARM64=true


To configure build for 32bit arm cpu arm7 (armeabi-v7a):
cmake -S ../.. -B . -DCMAKE_BUILD_TYPE=Release -DTARGET_OS_ANDROID=true, -DQT_VERSION=6_6_3 -DARM32=true

Compile Build:
make -j$(nproc)

====== ANDRROID ISSUES =======

1.) Newer Galaxy devices will break in java assembly when screen is touched
When debugging on galaxy A8 tablet and galaxy flip phone 5G 
There seems to be issues with touching the screen causing gralloc4 message stream and sometimes a break somewhere is Java assembly in QCreator
I have opened a bug QTBUG-119061


2.) Qml Prompts when debugging
NLC does not use Qml
 In Projects->Selected Android Kit->Run
	set QML Debugger: to Disabled
 In Projects->Selected Android Kit->Build
	set QML debugging and profiling to Disable
