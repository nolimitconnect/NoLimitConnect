# Contributing

NoLimitConnect welcomes casual contributors and experienced developers.
You do not need to be a maintainer to help.

## Quick Start Paths

### I have 30 minutes

1. Pick a docs issue or typo from [Issues](https://github.com/nolimitconnect/NoLimitConnect/issues).
2. Edit files under docs/.
3. Open a small PR.

Start here: [First Contribution (30 minutes)](developer-docs/first-contribution.md)

### I can do regular development

1. Read [Developer Docs](developer-docs/README.md).
2. Follow [Contributing Workflow](developer-docs/contributing.md).
3. Pick a bug or small feature and open a PR.

## Areas Where Help Is Most Useful

- Documentation and onboarding clarity
- Small reproducible bug fixes
- Build and packaging reliability
- Test and CI stability

## Project Principles

- No ads, no analytics, no tracking.
- Privacy-first and user-controlled.

## Maintainer-Level Help (Optional)

- Project representation and community moderation
- Bug triage, issue grooming, and release coordination
- Documentation, onboarding, and contributor support
- Technical leadership across roadmap planning and delivery

Any level of involvement is welcome.
