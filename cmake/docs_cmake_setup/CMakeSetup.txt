No Limit Connect maintains multiple build systems
1.) Visual Studio solution
2.) Qt .pro qmake system
3.) Cmake

GOALS:
1.) Setup cmake for a replacement for Visual Studio solution and Qt qmake system then drop support for the other build systems
2.) Cmake builds must be compatible with gitlab continuous integration system
3.) camke system should have a minimum number of parameters that need to be passed. 
	preferably there would be only 2 parameters, Release/Debug and TARGET_OS_WINDOWS/TARGET_OS_LINUX/TARGET_OS_ANDROID
	
NOTES:
Cmake is a mess to build with. 
1.) Generators - There are many options. I am attempting to use Ninja but this may change
2.) Configurations, Paths and variables for CMake are many methods and configurations

There seem to be many options to setup enviroment variables for cmake build system
1.) pass parameters on command line to cmake. Not a great option. 
	User has to reasearch what is the correct parameters
2.) Set enviroment varables then read the enviroment variables in cmake.
	A decent way and some are required like QTDIR but a pain if the variables need to change for each build type
3.) CMakePresets.json Is the preferred way but has complications
4.) Pass just minimum parameters to cmake. Directories and other cmake variables are hard coded into nolimitapp/cmake/cmake_helpers/OsDetect.cmake. Not ideal but is easiest to create/debug. 


To avoid the massive time sink of supporting many configureations only these configurations are currently supported.

1.) Android built on Windows Host OS - aarch64 arm is the only supported CPU
2.) Windows built on Windows Host OS - x64 is the only supported CPU
3.) Android built on Linux Host OS - aarch64 arm is the only supported CPU
4.) Linux built on Linux Host OS - x64 is the only supported CPU
5.) Windows built on Linux Host OS - x64 is the only supported CPU

Only CMake 3.20.0 or higher is supported
Only one Qt version is supported .. at time of this writing the Qt version is 6.3.2 and will be indicated by <Qt Version> in documentation
Only one Android setup is supported. assumes all tools and android ndk have been setup by QCreator
Assumes if host os is linux the Android tools path is $HOME/Android 
Assumes if host os is windows the Android tools path is F:/Android


================= Compilation Tools ==================
See setup instructions in nolimitapp/nolimitlib/docs_for_building/
 
=== JDK ===
If you plan to compile for android then install jdk 11 to $HOME/Android
set JAVA_HOME enviroment variable to jdk path
Example export JAVA_HOME=$HOME/Android/dk-11.0.13+8
If compiling on Windows host os substitute F: (or whatever drive you do builds on) for $HOME in path


=== NASM ===
NASM assembler can be installed from
https://www.nasm.us/ 
or if on Linux host os use sudo apt-get install nasm
NASM is needed mostly for ffmpeg video decoders when built for Windows or Linux. Android uses NDK cross compiler clang for assembly
add to path the NASM executable directory

=== Visual Studio 2019 ===
Visual Studio 2019 Community Edition should be installed if using Windows as host
This is optional but currently is preferred - The goal is to use windows cross compile with Qt in later updates and the need for Visual Studio removed

=== QT ===
Qt can be installed from 
https://www.qt.io/download
The open source version is preferred for No Limit Connect which is an open source project
Be sure to install Qt 6.3.2 and Android Developement, CMake and Ninja

=== CMake ===
=== windows host os ===
recommended for Windows Host OS
recommend download cmake from https://cmake.org/download/ and install instead of using the qt version of cmake
I seems to solve some issues like LINK : fatal error LNK1104: cannot open file 'kernel32.lib' ninja: build stopped: subcommand failed.

else if desired
option use the cmake installed by qt
add to path <Qt Install directory>/Tools/CMake_64/

=== Linux host os ===
cmake --version .. if less than 3.20.0 or not installed then
sudo apt-get remove cmake
then either checkout and build from source or in ubuntu install cmake from snap gives version 3.26.3 or higher
sudo snap install cmake --classic
or 
add to path <Qt Install directory>/Tools/CMake_64/

================= Test Build Enviroment ==================
=== minimal apps for configuration testing ===
=== You can test your configuration with sandboxqt minimal test apps ===

====== analog clock ======
analogclock - a simple Qt analog clock program copied from qt examples
build/run this first and you can determine if qt is setup correctly

option 1.) Build using cmake from command window

=== if building using Windows as host os ===
f:\
cd F:/nolimitapp/nolimitgui/sandboxqt/AnalogClockAppFromExamples/analogclock/build/cmake
cmake  -S ../../.. -B . -DCMAKE_BUILD_TYPE=Debug -DTARGET_OS_WINDOWS=true

=== if building using Linux as host os ===
cd ~/nolimitapp/nolimitgui/sandboxqt/AnalogClockAppFromExamples/analogclock/build/cmake
cmake  -S ../../.. -B . -DCMAKE_BUILD_TYPE=Debug -DTARGET_OS_LINUX=true

option 2.) Build using cmake from QCreator
open sandboxqt/AnalogClockAppFromExamples/CMakeLists.txt and configure and build

option 3.) Build using QCreator qmake gui.pro file
open sandboxqt/AnalogClockAppFromExamples/gui.pro and configure and build



====== Assembly1234TestApp ======
Assembly1234TestApp - A minimal Qt app that compiles and links assembly and c files as a static library
If you can build/run this app then most likely you can compile/run No Limit Connect
=== Host OS Windows Instructions ===

option 1.) Build using cmake from command window

=== if building using Windows as host os ===
f:\
cd F:/nolimitapp/nolimitgui/sandboxqt/Assembly1234TestApp/Assembly1234QtApp/build/cmake
cmake  -S ../.. -B . -DCMAKE_BUILD_TYPE=Debug -DTARGET_OS_WINDOWS=true
or 
cmake  -S ../.. -B . -DCMAKE_BUILD_TYPE=Debug -DTARGET_OS_ANDROID=true -DCMAKE_TOOLCHAIN_FILE=../../cmake/cmake_helpers/AndroidOnWindowsCrossCompile.cmake

=== if building using Linux as host os ===
cd ~/nolimitapp/nolimitgui/sandboxqt/Assembly1234TestApp/Assembly1234QtApp/build/cmake
cmake  -S ../../.. -B . -DCMAKE_BUILD_TYPE=Debug -DTARGET_OS_LINUX=true

option 2.) Build using cmake from QCreator
open sandboxqt/Assembly1234TestApp/CMakeLists.txt and configure and build

option 3.) Build using QCreator qmake xxx.pro files
open sandboxqt/Assembly1234TestApp/Assembly1234QtLib.pro and configure and build
open sandboxqt/Assembly1234TestApp/Assembly1234QtGui.pro and configure and build
